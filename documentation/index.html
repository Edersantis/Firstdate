HTML Structure - top

Template's layout is responsive, maximum width is 1170px with 1 main center column that can be divided into a 1/12 grid. This landing page uses the Twitter Bootstrap, HTML5 and CSS3.

section .section-grey
Section with grey background. You can have as many sections as you need.

Section with background in color the same as the main site's color. You can have as many sections as you need.

Minimalistic footer with logo and go to top button.

HTML structure

Permatex's HTML code is properly commented. We believe this will significantly increase your productivity while you make changes to the template. We've marked the beginning of all important elements so that you can easily modify or remove them without compromising the HTML's integrity and validity.
For production environments we recommend removing the HTML comments. If your HTML editor of choice can't remove them automatically you can easily clean your code using one of many free online tools such as HTML Tidy Online.

Themes affects the overall look of the whole template. You can use only one theme at a time. Pomegranate is the default one and it's defined around line #21 in all HTML files. If you want to use the color-picker to create a new unique theme simply open any theme file, replace the two color codes with new ones and use that file as the templates theme.

Main, top menu - top

Main menu is defined around line #47. Just add new LI elements with A elements in them or edit one of existing ones to configure the menu.
Mobile menu (a SELECT element) is auto-generated based on the "regular" menu. If you want to disable the auto-generate feature just remove the "data-autogenerate" property and enter the OPTION elements in it yourself having their value set to the link URL.
Menu on the front page is used to scroll to sections on that page but there's no reason you can't mix "regular" and anchor links. Just remember to put the "smoothscroll" class on links you want to have the "smooth" and slow scroll effect on. Links to other pages don't need that class.

JavaScript - top

All JS code is jQuery based. The only file you should be editing is "/js/common.js".


Configuring the contact form script
If you're having problems with the contact form please read the 5 seconds PHP setup first in this doc. There's no need to change anything in the JS/HTML unless you want to add/remove form elements.
Validation and AJAX form handling is defined and configured in "common.js". We've used one of the best validation scripts available - jQuery validation plugin which can be easily configured to validate any form. Lines #167-170 in "common.js" define the rules for input fields. Most of them are only "required" while the email field also needs to be a valid email address. Line #172 (messages) defines the error messages user gets when the form is not properly filled.
Once the form is submitted via AJAX response alerts are configured in the "contactFormResponse()" function. You'll probably want to change the message for successful sending which is defined at line #194.


Switching between MailChimp and "plain" newsletter subscription form
Find the form with id "newsletterform" around line #336. Set its "data-mailchimp" attribute to "true" if you want to use MailChimp or to "false" for file-based newsletter subscription.


Configuring the newsletter subscription form
If you're having problems with the newsletter subscription form (MailChimp or plain) please read the PHP setup first in this doc. There's no need to change anything in JS unless you want to change the response script gives to the user. Open "common.js" file which you'll find in the JS directory. The code you need starts on line #203 for plain newsletter and on #228 for MailChimp. You can change the single-quoted text in each line that starts with "alert". For instance if you change the text on line #221 ("Thank you for subscribing to our newsletter! We will not abuse your address.") you'll change the "thank you" text users see after a successful subscription.


Configuring the Twitter script
Please note that besides setting the feed details in JS you must also configure it in PHP. See PHP configuration in section below.
Open "common.js" file which you'll find in the JS directory. On line #56 there's a piece of code that contains "username: 'envato',". Simply replace "envato" with the Twitter username whose tweets you want to show and you're done! Don't forget the single quotes around the username.
For further details on how to configure the Twitter script please see tweet.seaofclouds.com.


Other JS
All JS is properly commented so changing any parameters is easy. Just edit the predefined values. Ie to make the slider rotate faster lower the numbers defined on line #52 which inits the slider.

PHP Files - top

Twitter Feed

Due to new Twitter API rules you have to create an application in order to show custom Twitter feeds. The process is very simple and free. Go to dev.twitter.com and login with your Twitter account (any account will do). Click "My applications" under your account's avatar and then click "Create a new application". Application details are really not important, just fill them in. After the app is created click "Create my access token" (blue button on the page bottom). Then open _twitter_feed.php found in the site's root folder and copy/paste 4 strings in appropriate variables. Please mind the single quotes; they have to be around the string.

Contact form

Permatex includes a versatile AJAX driven PHP script that powers the contact form.

5 seconds setup
Locate the file "_contact-us.php" which you'll find in the root directory of your chosen color theme. On line 9 there's a variable called $myEmail fill it with your email address, ie "$myEmail = 'john@gmail.com';". You're done!
Test the script by entering some data into the contact form and you should receive an email momentarily.

Autoresponder setup
By default autoresponder is disabled. Enable it by setting the $autorespond variable on line #15 to "true". Whenever a client submits the form he/she will get a thank you email. Subject is defined on line #18 in variable $autorespondSubject while the email body can be set on line #21 in variable $autorespondMsg.

5 minutes setup
Besides the email address you can also configure the email's subject. You'll find the variable "$emailSubject" on line 11 of "_contact-us.php" script.
If you add or remove some form fields there is no need to change anything in the script. It will email you all the fields regardless of their number.
All other parameters of this script are configured in JavaScript. You can read more about it in the JavaScript section of this manual.

Newsletter script

Permatex comes with MailChimp driven newsletter signup and a simple AJAX driven PHP script that enables you to collect email addresses from your visitors who wish to subscribe to your newsletter.

Script requirements
PHP 5.2 or an older version of PHP with JSON extension. Privileges to modify write-access (chmod) on one txt file - plain script only. Ability to send emails via PHP's "mail()" function - plain script only.
These requirements are met by default in almost all hosting environments that support PHP.

Setting up the MailChimp newsletter script
Open "_newsletter-subscribe-mailchimp.php". Lines #9 and #11 contain the only things you have to change. MailChimp API key can be obtained in your MC account under "Account - API keys". Carefully copy/paste the API key into single quotes on line 9. Line #11 contains the name of MC list you want the people to subscribe to. If needed create a new list in your MC acc.

Setting up the plain newsletter script
5 seconds setup
Locate the file "subscribed-emails.txt" which you'll find in the root www directory of template. Chmod it to 777 so that PHP can write the emails in it. You're done!
Test the script by entering an email in "subscribe" box on your site, hit enter and it should say "Thank you for subscribing".

5 minutes setup
All PHP code is located in a single file "_newsletter-subscribe.php". There are 2 variables that you can configure. The first one, "$emailsFile" (line 10) defines the path to TXT file in which PHP stores the emails and checks for duplicates. PHP has to have read and write access to this file so be sure to chmod it to 777 or a similar value that will give PHP the noted access rights. Second variable "$myEmail" (line 15) is an optional parameter which enables you to receive an email each time someone subscribes to your newsletter. Set this variable to your email address, ie: "myemail@gmail.com". If you don't want to use this feature just leave it blank.
All other parameters of this script are configured in JavaScript. You can read more about it in the JavaScript section of this manual.
